Custom Train Config Handbook

custom 模式读取 custom_train_config.json。本页与 docs/custom-train-config.md 同步，作为 Web UI 内嵌参数手册。

建议流程：先跑默认配置，再每轮只改 1~2 个参数，并用不同 run_name 做对比。

高影响参数

字段作用推荐起点
episodes训练总局数30000
model_type网络结构adaptive_cnn 或 hybrid
learning_rate学习率1e-4
batch_size每次更新采样数128
epsilon_start/end/decay_steps探索策略1.0 / 0.03 / 200000
replay_capacity回放池容量50000 起
target_update_interval目标网络同步步数2000
run_name输出目录名每次实验唯一

模型参数

model_type

  • small_cnn：固定尺寸输入，不适合 curriculum/random_board。
  • adaptive_cnn：支持可变尺寸，通用推荐。
  • hybrid：可变尺寸 + 更强泛化，计算略重。

local_patch_size

仅 hybrid 有效，必须正奇数（如 11）。

探索参数

{
  "epsilon_start": 1.0,
  "epsilon_end": 0.03,
  "epsilon_decay_steps": 200000
}
  • 收敛太慢：适度减小 epsilon_decay_steps。
  • 后期质量差：适度增大 epsilon_decay_steps。

回放与更新

{
  "replay_capacity": 50000,
  "min_replay_size": 3000,
  "batch_size": 128,
  "train_frequency": 4,
  "target_update_interval": 2000
}
  • 内存紧张：先降 replay_capacity，再降 batch_size。
  • 训练抖动：先降 learning_rate，再评估是否调大 target_update_interval。

环境参数（env）

{
  "env": {
    "board_size": 14,
    "difficulty": "normal",
    "mode": "classic",
    "max_steps_without_food": 196,
    "enable_bonus_food": false,
    "enable_obstacles": false,
    "allow_leveling": false,
    "seed": 42
  }
}
  • 入门建议：board_size=10~14，复杂机制先关闭。
  • max_steps_without_food 常设为 board_size^2。

奖励参数（reward_weights）

{
  "reward_weights": {
    "alive": -0.01,
    "food": 1.0,
    "bonusFood": 1.5,
    "death": -1.5,
    "timeout": -1.0,
    "levelUp": 0.2,
    "victory": 5.0,
    "foodDistanceK": 0.4
  }
}

优先调这 3 个：death、alive、foodDistanceK。

课程学习 / 随机地图 / 并行

  • curriculum 与 random_board 互斥。
  • 开启任意一个时，model_type 必须是 adaptive_cnn 或 hybrid。
  • 并行建议先关闭，串行稳定后再开启：
{
  "parallel": {
    "enabled": false,
    "num_workers": 4,
    "weight_sync_interval_steps": 512,
    "actor_device": "cpu"
  }
}

校验与排错

推荐先估算，再正式训练：

uv run snake-rl estimate --scheme custom --custom-config custom_train_config.json
  • “可变尺寸不支持”错误：通常是 small_cnn + curriculum/random_board 组合。
  • local_patch_size 报错：改为正奇数。
  • 曲线不涨：优先检查 epsilon_decay_steps、learning_rate、奖励权重。
